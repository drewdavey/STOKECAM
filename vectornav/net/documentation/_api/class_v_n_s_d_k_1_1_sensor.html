class Sensor¶

Serial Connectivity

void Connect(String *portName, UInt32 baudRate)¶

Opens the serial port at the specified baud rate. This does not verify any connectivity, and as such only validates that the serial port is available available for opening.

void AutoConnect(String *portName)¶

Opens the serial port, scanning all possible baud rates until the sensor is verified connected. This performs a verifySensorConnectivity at each possible baud rate.

bool VerifySensorConnectivity()¶

Sends a ReadRegister for the Model register. Returns true if a valid response is received, otherwise returns false.

String *ConnectedPortName()¶

Get the port name of the open serial port. If no port is open, will return std::nullopt.

uint32_t ConnectedBaudRate()¶

Get the baud rate at which the serial port is opened. If no port is open, will return std::nullopt.

void ChangeBaudRate(UInt32 baudRate)¶

Sends a Write Register for the new baud rate to the sensor and reopens to the serial port under the new baud rate. Will retry on failure.

void ChangeHostBaudRate(UInt32 baudRate)¶

Changes the host (computer) baud rate without commanding a change to the VectorNav unit’s baud rate. Use with caution.

void Disconnect()¶

Disconnect from the sensor.

Accessing Measurements

bool HasMeasurement()¶

Checks to see if there is a new measurement avialable on the MeasurementQueue.

Nullable<CompositeData> GetNextMeasurement()¶

Gets (and pops) the front of the MeasurementQueue.

Parameters

block – If true, wait a maximum of getMeasurementTimeoutLength for a new measurement.

Nullable<CompositeData> GetMostRecentMeasurement()¶

Gets the back (most recent) of the MeasurementQueue, popping every measurement in the queue.

Parameters

block – If true, wait a maximum of getMeasurementTimeoutLength for a new measurement.

Sending Commands

void ReadRegister(Registers::Register *reg)¶

Sends a Read Register command to the unit to poll register values. This is always a blocking call.

void WriteRegister(Registers::Register *reg)¶

Sends a Write Register command to the unit to set register values. This is always a blocking call.

void WriteSettings()¶

Sends the Write Settings command to the unit and blocks on the unit’s confirmation.

void Reset()¶

Sends a Reset command to the unit and blocks on the unit’s confirmation. After confirmation, it sleeps for resetSleepDuration and verifies sensor connectivity, calling autoConnect if failed.

void RestoreFactorySettings()¶

Sends a Restore Factory Settings command to the unit, blocks on the unit’s confirmation, reopens serial at the unit’s default baud rate, sleeps for resetSleepDuration, then verifies unit connectivity. If THREADING_ENABLE, resets the Listening Thread.
